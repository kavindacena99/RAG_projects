Software Engineering is the systematic process of designing, developing, testing, deploying, and maintaining software in a structured and reliable way.

It is not only about writing code. It also includes planning, teamwork, documentation, testing, quality assurance, and maintenance.

Goal of Software Engineering:
To build software that is:

Correct
Reliable
Scalable
Maintainable
Secure
Efficient

Why Software Engineering is important:
Without a proper process, software projects can become difficult to manage, full of bugs, late, and hard to maintain.

Main phases of Software Development Life Cycle (SDLC):

Requirement Analysis
Understand what the user or client needs.

Questions:

What problem are we solving?
Who are the users?
What features are required?
System Design
Plan how the system will work.

Includes:

Architecture design
Database design
UI/UX design
API design
Implementation / Coding
Write the actual code based on the design.
Testing
Check whether the software works correctly.

Types of testing:

Unit Testing
Integration Testing
System Testing
Acceptance Testing
Deployment
Release the software for users.
Maintenance
Fix bugs, improve features, and update the system after release.

Agile software engineering:
Agile is a popular way of building software in small, iterative steps.
Instead of building everything at once, teams deliver features gradually and improve based on feedback.
